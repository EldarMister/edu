# EDU POS

EDU POS - POS-система для кафе и ресторана. Проект закрывает основной рабочий цикл заведения: официант выбирает стол и отправляет заказ, кухня получает заказ в реальном времени, заказ проходит статусы готовки и подачи, после этого официант принимает оплату и закрывает стол.

Проект состоит из двух приложений:

- `backend` - NestJS API, Prisma, PostgreSQL, Socket.IO.
- `frontend` - React PWA для официанта, кухни, администратора и владельца.

## Возможности

### Официант

- Авторизация по телефону и паролю.
- Проверка активной смены.
- Старт и завершение смены.
- Выбор зала и стола.
- Адаптивная сетка столов:
  - до 10 столов - 3 крупные колонки;
  - больше 10 столов - 4 компактные колонки.
- Цветовые статусы столов.
- Выбор блюд по категориям.
- Поиск блюд.
- Корзина с количеством, ценой и комментариями.
- Создание нового заказа.
- Добавление блюд в существующий активный заказ.
- Просмотр активных заказов.
- Получение real-time уведомлений от кухни.
- Действия по готовому заказу: забрал с кухни, подал, перейти к оплате.
- Оплата заказа через QR или наличные.
- Формирование и печать чека.
- Скрытые скроллбары в мобильном интерфейсе официанта.

### Кухня

- Отдельный интерфейс кухни.
- Получение новых заказов в real-time.
- Вкладки заказов: новые, в работе, готовые, отказанные.
- Принятие заказа в работу.
- Отметка заказа как готового.
- Отказ по всему заказу с причиной.
- Отказ по отдельной позиции с причиной.
- Таймер ожидания заказа.
- Уведомление официанта о статусе заказа.

### Администратор и владелец

- Единая админ-панель с доступом по ролям.
- Просмотр заказов, фильтрация и поиск.
- Управление залами и столами.
- Управление категориями и блюдами.
- Управление сотрудниками.
- Статистика для владельца.
- Real-time обновление заказов и статусов столов.

### Общая логика

- JWT-авторизация.
- Разделение доступа по ролям: `WAITER`, `KITCHEN`, `ADMIN`, `OWNER`.
- Автоматический редирект пользователя в интерфейс своей роли.
- Access/refresh token flow.
- Socket.IO события для заказов, уведомлений и столов.
- PWA-манифест.
- Логотип из `frontend/public/icon1.png`.
- Номера заказов отображаются без ведущих нулей: `№000006` -> `№6`.

## Переменные окружения

### Backend: `backend/.env`

```env
DATABASE_URL=postgresql://USER:PASSWORD@HOST:PORT/DBNAME?schema=public

JWT_ACCESS_SECRET=change-me-access-secret
JWT_REFRESH_SECRET=change-me-refresh-secret
JWT_ACCESS_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d

PORT=3000
CORS_ORIGIN=http://localhost:5173

REDIS_URL=
```

`REDIS_URL` сейчас опционален. При пустом значении приложение работает как один Socket.IO instance.

### Frontend: `frontend/.env`

```env
VITE_API_URL=http://localhost:3000
```

Во фронтенде `/api` добавляется автоматически в `src/lib/api.ts`.

## Основной сценарий работы

1. Официант входит в систему.
2. Официант начинает смену в профиле.
3. Официант выбирает зал и стол.
4. Официант выбирает блюда и отправляет заказ на кухню.
5. Кухня видит новый заказ в real-time.
6. Кухня принимает заказ.
7. Официант видит обновлённый статус заказа.
8. Кухня отмечает заказ готовым или отказывает в заказе/позиции.
9. Официант получает уведомление.
10. Официант нажимает "Забрал с кухни".
11. Официант нажимает "Вынес гостям".
12. Официант переводит заказ к оплате.
13. Официант принимает оплату.
14. Система формирует чек.
15. Заказ закрывается, стол освобождается.

## Основные backend-модули

- `auth` - вход, refresh token, JWT strategy.
- `orders` - создание заказа, добавление блюд, статусы кухни и официанта.
- `payments` - приём оплаты и получение чека.
- `kitchen` - список заказов кухни.
- `waiter-shifts` - смены официанта и статистика смены.
- `halls`, `tables`, `categories`, `dishes` - публичные справочники.
- `admin` - управление каталогом, заказами, персоналом и статистикой.
- `realtime` - Socket.IO gateway.

## Real-time события

События используются для синхронизации интерфейсов без ручного обновления страницы:

- новый заказ для кухни;
- изменение статуса заказа;
- изменение статуса стола;
- уведомления официанту;
- готовность заказа;
- отказ по заказу или позиции.
