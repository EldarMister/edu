# Вкусно • POS

POS-система для кафе/ресторана: официант принимает заказы со смартфона, кухня получает их в реальном времени, администратор и владелец (на следующих этапах) управляют процессами и видят статистику.

> Разработка ведётся **поэтапно** (см. `ТЗ/Условие перед началом.md`).
> Текущий объём: **Этапы 1–4** — фундамент, единая авторизация, официантская часть, кухонная часть и real-time связка между ними. Админка и панель владельца — на следующих этапах.

## Стек

**Backend:** Node.js · TypeScript · NestJS · Prisma · PostgreSQL · JWT + RBAC · Socket.IO
**Frontend:** React · TypeScript · Vite · Tailwind CSS · React Router · Zustand · TanStack Query · Socket.IO Client · PWA

## Структура

```
edu pos/
├── backend/      # NestJS API + Socket.IO gateway + Prisma
├── frontend/     # React PWA (официант + кухня)
├── designe/      # Дизайн-референсы (авторизация, столы, кухня)
└── ТЗ/           # Техническое задание
```

## Требования

- Node.js 20+ (проверено на 24)
- PostgreSQL 14+ (своя БД — данные подключения в `backend/.env`)

## Запуск

### 1. Backend

```bash
cd backend
cp .env.example .env        # затем впишите DATABASE_URL и JWT-секреты
npm install
npm run prisma:generate
npm run prisma:migrate       # создаст таблицы в вашей БД
npm run prisma:seed          # тестовые пользователи, зал, столы, меню
npm run start:dev            # http://localhost:3000
```

### 2. Frontend

```bash
cd frontend
cp .env.example .env         # VITE_API_URL по умолчанию http://localhost:3000
npm install
npm run dev                  # http://localhost:5173
```

## Тестовые пользователи (после seed)

| Роль       | Телефон      | Пароль     |
|------------|--------------|------------|
| Официант   | +70000000001 | waiter123  |
| Кухня      | +70000000002 | kitchen123 |
| Админ      | +70000000003 | admin123   |
| Владелец   | +70000000004 | owner123   |

Вход выполняется по **номеру телефона + паролю**. После входа система определяет роль и перенаправляет в нужный интерфейс.

## Сценарий проверки (этап 5)

Официант: вход → выбор стола → блюда → «Отправить на кухню» → кухня видит заказ в реальном времени → «Принять» → официант видит статус → «Готово» → официант получает уведомление → «Забрал» → оплата (QR/наличка) → чек → закрытие → стол свободен.
